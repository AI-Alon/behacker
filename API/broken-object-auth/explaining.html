Broken Object-Level Authorization (BOLA / IDOR)

What is it?

BOLA (also called IDOR — Insecure Direct Object Reference) is the #1 API vulnerability according to OWASP API Security Top 10. It occurs when an API uses a user-supplied identifier (ID, UUID, filename) to access a resource without verifying that the currently authenticated user is allowed to access it.

How it works

Vulnerable endpoint:

GET /api/orders/1001        → returns Alice's order
GET /api/orders/1002        → returns Bob's order (Alice can access this!)

The server fetches the order by ID but never checks order.userId === session.userId.

Vulnerable code:

app.get('/api/orders/:id', async (req, res) => {
  const order = await Order.findById(req.params.id);
  // ❌ No ownership check!
  res.json(order);
});

Secure code:

app.get('/api/orders/:id', async (req, res) => {
  const order = await Order.findOne({
    id: req.params.id,
    userId: req.session.userId  // ✅ ownership enforced at DB level
  });
  if (!order) return res.status(403).json({ error: 'Forbidden' });
  res.json(order);
});

Variants

  • BOLA (read): Access another user's data by guessing their ID
  • BOLA (write/delete): Modify or delete another user's resources
  • BFLA: Call admin-level functions as a regular user
  • Indirect reference: Predict filenames, sequential IDs, UUIDs

Impact

  • Read other users' PII, financial records, private messages, documents
  • Delete or modify other users' accounts or orders
  • Escalate privileges by calling admin endpoints

Real-world examples

  • Uber (2016): Could access any driver's trip history by changing trip ID
  • Instagram: Could retrieve private photos by guessing media IDs
  • T-Mobile: Customer account data accessible by changing account number in API

Defense

  1. Never trust client-supplied IDs alone — always join with the authenticated user's ID
  2. Use non-sequential IDs (UUIDs v4) to slow down enumeration (but this is not a fix — authorization is)
  3. Test with two accounts: verify user A cannot access user B's resources
  4. Implement automated BOLA tests in your API test suite
