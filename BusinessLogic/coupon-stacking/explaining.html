Coupon Stacking

What is it?

Coupon stacking exploits a server that validates coupons individually (is this a valid code?) but doesn't track which codes have already been applied to the current order. The same code can be applied multiple times, or incompatible codes can be combined, to drive the price to zero.

Attack patterns

  1. Duplicate codes: Apply SAVE10 five times → 50% off (or 100% off if applied enough)
  2. Incompatible stacking: Combine HALF50 + VIP25 + LOYALTY15 → effectively free
  3. Race condition stacking: Apply the same single-use code concurrently in two browser tabs

Code vulnerability

// ❌ Vulnerable: checks if code is valid but not if already used
app.post('/apply-coupon', (req, res) => {
  if (VALID_CODES.includes(req.body.code)) {
    order.discount += DISCOUNTS[req.body.code];
  }
});

Defense

-- Track applied codes per order
INSERT INTO order_coupons (order_id, code) VALUES (?, ?)
  ON CONFLICT DO NOTHING;  -- prevents duplicates at DB level

-- Before applying, check
SELECT COUNT(*) FROM order_coupons WHERE order_id=? AND code=?;

Define explicit combination rules: max one percentage-off code, max total discount cap (e.g. 60%), and enforce them server-side.
