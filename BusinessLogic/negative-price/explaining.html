Negative Price Attack

What is it?

A negative price attack exploits missing server-side validation of numeric inputs (quantity, price, discount). By submitting negative quantities or overflowing discount combinations, the order total becomes negative — causing the system to issue a refund rather than charge the customer.

How it works

The server multiplies item_price × quantity × (1 - discount). If the server doesn't validate quantity > 0 and total >= 0:

  • quantity = -5 → total = -$249.95 → "refund" issued
  • discount = 1.5 (150%) → total = -$50 → same result

Real-world examples

  • Booking platforms: Negative room nights for refunds
  • E-commerce: Negative quantity items to drain gift card balances
  • Subscription services: Negative billing cycle adjustments

Defense

// Server-side validation
if (quantity <= 0) throw new Error('Quantity must be positive');
if (total < 0) throw new Error('Order total cannot be negative');
// Always recompute price from trusted product catalog
const total = catalog[productId].price * quantity * (1 - validatedDiscount);

Never trust client-supplied prices, quantities, or totals. Recompute everything server-side from authoritative data.
