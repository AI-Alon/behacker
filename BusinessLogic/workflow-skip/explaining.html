Workflow Skip (Forced Browsing / Step Bypass)

What is it?

Multi-step workflows (checkout, account verification, 2FA) enforce a logical sequence in the UI. But if the server doesn't independently verify that prerequisite steps were completed, users can navigate directly to any step URL and skip earlier ones — including payment.

Attack

Navigate directly to GET /checkout/step/4/confirm without completing steps 2 (address) and 3 (payment). The server processes the order confirmation anyway because it checks only whether the current step is valid, not whether prior steps were done.

Real-world impact

  • Skip payment → receive goods for free
  • Skip identity verification → change email/phone without proof
  • Skip 2FA step → access account with password only
  • Skip terms acceptance → bypass legal agreements

Vulnerable server-side code

// ❌ Only checks if step 4 endpoint exists — no prerequisite validation
app.post('/checkout/step/4/confirm', (req, res) => {
  createOrder(req.session.cart);  // No check: was payment completed?
});

Defense

// ✅ Server-side state machine
app.post('/checkout/step/:step', (req, res) => {
  const step = parseInt(req.params.step);
  const session = req.session;
  // Check all prerequisites
  if (step >= 2 && !session.addressComplete) return res.redirect('/checkout/step/2');
  if (step >= 3 && !session.paymentComplete) return res.redirect('/checkout/step/3');
  if (step >= 4 && !session.paymentVerified) return res.status(403).send('Payment required');
  // proceed
});

Enforce step completion in session state, not URL parameters. Never rely on the client navigating correctly.
