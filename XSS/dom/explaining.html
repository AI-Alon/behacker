DOM-Based XSS — How It Works

Normal rendering (safe)

The page reads the URL hash and displays the name as text:

const name = new URLSearchParams(location.hash.slice(1)).get("name");
welcomeEl.textContent = `Welcome, ${name}!`;

Output: Welcome, <script>alert(1)</script>! ← shown as literal text.

Vulnerable rendering (hash → innerHTML)

const name = new URLSearchParams(location.hash.slice(1)).get("name");

// VULNERABLE: attacker-controlled value written to the DOM
outputBox.innerHTML = `<p>Welcome, <strong>${name}</strong>!</p>`;

If the URL is: index.html#name=<img src=x onerror="alert('XSS')">

The browser reads the hash, inserts the raw value into innerHTML, and executes the onerror handler.

Why "DOM-based" is different

The server never sees the payload.

The #fragment part of a URL is not sent in HTTP requests.

Request:  GET /index.html          ← no hash, server logs show nothing
Browser:  reads location.hash      ← payload processed entirely client-side

This means:

  • Server-side output encoding doesn't help
  • Web Application Firewalls (WAFs) can't see the payload
  • Traditional server-side security tools are blind to it

Attack flow

  1. Attacker crafts a link:

index.html#name=<svg onload=alert(document.cookie)>

  1. Victim clicks the link
  2. Browser loads the page normally (server sees nothing suspicious)
  3. Client-side JS reads the hash and writes it to innerHTML
  4. Payload executes in the victim's browser

Common DOM sinks (dangerous functions)

element.innerHTML = userValue;       // ← most common
element.outerHTML = userValue;
document.write(userValue);
eval(userValue);
setTimeout(userValue, 0);
location.href = userValue;           // ← open redirect / JS URL

The Fix

// Safe: use textContent — never interprets value as HTML
welcomeEl.textContent = `Welcome, ${name}!`;

// If HTML structure is needed: sanitise first
import DOMPurify from "dompurify";
outputBox.innerHTML = DOMPurify.sanitize(`<p>Welcome, <strong>${name}</strong>!</p>`);

Never pass URL-derived values (hash, query string, referrer) to any DOM sink without sanitisation.
