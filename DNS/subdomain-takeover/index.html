Subdomain Takeover

Claiming an abandoned subdomain pointing to a deprovisioned cloud resource

When a service (GitHub Pages, Heroku, S3, Netlify) is deprovisioned but its DNS CNAME record is not removed, an attacker can register the same resource name on that platform and receive all traffic for that subdomain.

Step 2 — Check DNS Records
Step 3 — Claim the Resource

If the CNAME target is unclaimed, register it on the platform to take over the subdomain.
