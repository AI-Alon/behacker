DNS Rebinding

Bypassing the Same-Origin Policy via DNS TTL manipulation

DNS rebinding exploits a low TTL to swap what IP a domain resolves to mid-session. The browser trusts requests to attacker.com, but after rebinding the DNS to 127.0.0.1, the same origin now reaches the victim's localhost — bypassing the Same-Origin Policy.

Phase 1 — Victim visits attacker.com. DNS resolves to attacker's server (1.2.3.4). Page loads malicious JS.
Phase 2 — TTL expires (5s). Attacker changes DNS to resolve attacker.com → 127.0.0.1.
Phase 3 — JS makes XHR to attacker.com/admin. Browser resolves DNS again → gets 127.0.0.1. Request hits victim's local service.
Phase 4 — Response from localhost:8080/admin returned to attacker's JS. Same-origin policy bypassed.
