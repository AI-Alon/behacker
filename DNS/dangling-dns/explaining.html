Dangling DNS Record

What is it?

A dangling DNS record is a DNS entry (usually a CNAME) that points to an external resource that no longer exists. The record was never cleaned up after the resource was deleted. This is essentially subdomain takeover but discovered by scanning DNS records rather than subdomain enumeration.

Difference from subdomain takeover

  • Subdomain takeover: Discovered by finding subdomains and checking if CNAME targets are claimable
  • Dangling DNS: Same vulnerability, found by auditing the DNS zone directly for orphaned records

Common platforms with claimable resource names

  • GitHub Pages: <org>.github.io — claimable
  • Netlify: <site>.netlify.app — claimable
  • Heroku: <app>.herokuapp.com — claimable
  • AWS S3: <bucket>.s3.amazonaws.com — claimable (same region)
  • Azure: <app>.azurewebsites.net — claimable
  • Shopify: <store>.myshopify.com — claimable
  • Zendesk: <company>.zendesk.com — claimable

Impact

  • Serve phishing pages under the legitimate domain
  • Intercept OAuth redirect tokens (if subdomain is a registered redirect URI)
  • Cookie theft (cookies scoped to .example.com readable from subdomain)
  • Bypass email SPF/DMARC (send phishing from legitimate subdomain)
  • CSP bypass (subdomain may be in the allowlist)

Automated scanning workflow

# Export DNS zone
dig axfr example.com @ns1.example.com > zone.txt

# Extract CNAMEs
grep CNAME zone.txt | awk '{print $5}' > cname_targets.txt

# Check if targets are claimed
# subjack handles fingerprint matching per platform
subjack -w cname_targets.txt -t 100 -o results.txt -ssl

# Or nuclei with takeover templates
nuclei -l subdomains.txt -t ~/nuclei-templates/takeovers/

Prevention

  1. Use IaC (Terraform/Pulumi) — deleting a resource also removes its DNS record
  2. Quarterly DNS audits: compare all CNAME targets against active resource inventory
  3. Enable alerts when DNS records are added without a corresponding resource being created
  4. Before deprovisioning any cloud resource: git grep the resource name in DNS config, remove the record first
