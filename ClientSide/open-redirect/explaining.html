Open Redirect — How It Works

What is it?

An open redirect occurs when a web application redirects the user to a URL supplied by the user (via query parameter, form field, or header) without validating that the destination is on the same domain.

Vulnerable server-side code

// Node/Express — vulnerable
app.post("/login", (req, res) => {
  const { username, password } = req.body;
  if (validCredentials(username, password)) {
    const next = req.query.next || "/dashboard";
    res.redirect(next);   // ← no validation — blindly redirects
  }
});

Attack payload variations

# Basic external redirect
?next=https://evil.com/phishing-clone

# Protocol-relative — browser adds https: automatically
?next=//evil.com

# Domain confusion — looks like the real site in a quick glance
?next=https://securecorp.com.evil.com/login

# JavaScript URL — may execute code on the redirect page
?next=javascript:alert(document.cookie)

# Encoded to bypass naive filters
?next=https%3A%2F%2Fevil.com
?next=https:\/\/evil.com

Why victims trust open redirect URLs

The crafted link starts with the legitimate domain:

https://securecorp.com/login?next=https://evil.com/steal

Users and email security tools see securecorp.com and trust it. The redirect happens silently after login.

Chaining with other attacks

  • Phishing — post-login redirect to a credential-harvesting clone
  • OAuth token theft — redirect_uri validation bypass leaks access tokens
  • Password reset poisoning — redirect reset link to attacker domain
  • SSRF — server follows the redirect internally, reaching internal services

The Fix

const ALLOWED_HOSTS = ["securecorp.com", "www.securecorp.com"];

function isSafeRedirect(url) {
  // Allow relative paths (no protocol, no //)
  if (/^\/[^\/]/.test(url)) return true;
  try {
    const parsed = new URL(url);
    return ALLOWED_HOSTS.includes(parsed.hostname);
  } catch { return false; }
}

app.post("/login", (req, res) => {
  const next = req.query.next;
  const safe = isSafeRedirect(next) ? next : "/dashboard";
  res.redirect(safe);
});
