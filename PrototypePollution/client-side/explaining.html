Client-Side Prototype Pollution

What is it?

In JavaScript, every object inherits properties from Object.prototype. If an attacker can add a property to Object.prototype, that property becomes available on every plain object in the application — potentially bypassing access controls, enabling XSS, or triggering logic flaws.

How it happens

Vulnerable query string parsers and deep merge utilities process keys like <em>_proto</em>_ as object traversal paths:

// Vulnerable parser
parseQuery('?__proto__[isAdmin]=true')
// → effectively: Object.prototype.isAdmin = 'true'

// Now every object inherits isAdmin:
const user = { name: 'alice' };
console.log(user.isAdmin); // → 'true' (from prototype!)

Vulnerable deep merge

function merge(target, source) {
  for (const key of Object.keys(source)) {
    if (typeof source[key] === 'object') {
      merge(target[key], source[key]); // ← recurses into __proto__
    } else {
      target[key] = source[key];
    }
  }
}
merge({}, JSON.parse('{"__proto__":{"isAdmin":true}}'));

Known vulnerable packages (historical)

  • lodash < 4.17.5 — <em>.merge(), </em>.zipObjectDeep()
  • jquery < 3.4.0 — $.extend()
  • qs < 6.7.3 — query string parser
  • handlebars < 4.4.5 — template engine

Defense

// Option 1: Block prototype-polluting keys
function safeMerge(target, source) {
  for (const key of Object.keys(source)) {
    if (key === '__proto__' || key === 'constructor' || key === 'prototype') continue;
    // ...
  }
}

// Option 2: Use null-prototype objects for untrusted data
const config = Object.create(null);

// Option 3: Freeze Object.prototype
Object.freeze(Object.prototype);

// Option 4: Check own properties only in security checks
if (Object.hasOwn(user, 'isAdmin') && user.isAdmin) { ... }
