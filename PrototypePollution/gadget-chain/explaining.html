Prototype Pollution Gadget Chain

What is it?

Prototype pollution alone is not dangerous without a "gadget" — a code path that reads the polluted property and uses it in a sensitive way (setting innerHTML, passing to eval, using as a URL). A gadget chain connects the pollution source to a dangerous sink.

The chain

Source: vulnerable parser/merger
    ↓
Object.prototype.__proto__.innerHTML = "<script>alert(1)</script>"
    ↓
Gadget: widget renderer reads options.innerHTML
    ↓ (options has no own innerHTML, falls back to prototype)
el.innerHTML = "<script>alert(1)</script>"
    ↓
Sink: XSS executed

Common gadget types

  • innerHTML — Template renderer → XSS
  • src + onerror — Image loader → XSS
  • template — Handlebars/Pug renderer → SSTI → RCE
  • nodeType — DOM manipulation code → Logic bypass
  • isAdmin — Access control check → Privilege escalation
  • NODE_OPTIONS — Node.js spawn → RCE (server-side)

Historical CVEs

  • CVE-2019-10744 — lodash _.defaultsDeep() → XSS gadget via Handlebars
  • CVE-2020-28477 — immer library → prototype pollution
  • jQuery — $.extend(true, {}, ...) → pollution via deep merge

Defense

// Sanitize all HTML before setting innerHTML
el.innerHTML = DOMPurify.sanitize(options.innerHTML);

// Check own properties before using as HTML
if (Object.hasOwn(options, 'innerHTML')) {
  el.innerHTML = DOMPurify.sanitize(options.innerHTML);
}

// Use textContent instead of innerHTML where HTML is not needed
el.textContent = options.message;
