Directory Listing — How It Works

What is directory listing?

When a web server receives a request for a directory and finds no index.html (or index.php) inside it, it can either return a 403 Forbidden or — if misconfigured — return a full HTML page listing all files in the directory.

The misconfiguration

# Nginx — vulnerable
server {
  root /var/www/securecorp;
  autoindex on;       ← enables directory listing
}

# Apache — vulnerable
Options +Indexes      ← in .htaccess or httpd.conf

What an attacker finds

Browsing to /uploads/ reveals every uploaded file — including:

shell.php          ← a web shell that was already uploaded
users_export.csv   ← full user database dump
webshell.jpg       ← a PHP shell disguised as an image

Browsing to /.git/ exposes the git repository, allowing full source code extraction:

# Reconstruct source code from exposed .git/
git clone --mirror https://target.com/.git/ /tmp/repo
git checkout -- .

Discovery methods

# Gobuster — enumerate directories
gobuster dir -u https://target.com -w common.txt --add-slash

# Nikto — automatic misconfiguration scan
nikto -h https://target.com

# Manual — just append / to any guessed path
https://target.com/backup/
https://target.com/logs/
https://target.com/.git/

The Fix

# Nginx — disable directory listing globally
server {
  autoindex off;        ← default, but must be explicit
}

# Apache
Options -Indexes        ← in httpd.conf or .htaccess

# Belt-and-suspenders: place an empty index.html in every sensitive directory
echo "" > /var/www/securecorp/uploads/index.html
echo "" > /var/www/securecorp/backup/index.html

# Most importantly: move sensitive directories outside the web root entirely
# /var/backups/db/  instead of  /var/www/securecorp/backup/
