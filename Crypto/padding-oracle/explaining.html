Padding Oracle Attack

What is it?

A padding oracle attack exploits a server that reveals whether a decrypted ciphertext has valid PKCS#7 padding. This single bit of information — "valid" vs "invalid" padding — allows an attacker to decrypt any ciphertext block or forge arbitrary plaintext, without knowing the encryption key.

Background: AES-CBC and PKCS#7 padding

AES-CBC encrypts in 16-byte blocks. The last block is padded to fill 16 bytes:

  • 1 missing byte → \x01
  • 2 missing bytes → \x02\x02
  • 3 missing bytes → \x03\x03\x03
  • 16 missing bytes → \x10\x10...\x10 (full padding block)

During decryption, the server XORs the ciphertext block with the previous block (IV for block 1), then checks that the padding is valid. If not, it throws an error — that error is the oracle.

How decryption works (CBC)

Plaintext[i] = Decrypt(CT[i]) XOR CT[i-1]

By manipulating CT[i-1] (or the IV), we control the XOR that's applied to the raw decrypted bytes.

The attack in steps

Goal: Decrypt byte 15 (last byte) of block 1

  1. Set byte 15 of the IV to values 0x00–0xFF, sending 256 probes
  2. When the oracle returns "valid padding", Decrypted[15] XOR IV'[15] = 0x01
  3. Therefore: Intermediate[15] = 0x01 XOR IV'[15]
  4. Actual plaintext: Plaintext[15] = Intermediate[15] XOR IV_original[15]

Repeat for bytes 14, 13, ..., 0 (adjusting padding target each time).

Full block decryption requires at most 256 × 16 = 4,096 probes.

Forging arbitrary plaintext

Once intermediate values are known:

new_IV[i] = target_plaintext[i] XOR Intermediate[i]

The server will decrypt the ciphertext to exactly target_plaintext.

Example: forge a cookie from role=user to role=admin without the key.

Real-world instances

  • POODLE (2014): SSL 3.0 — forced CBC with padding oracle
  • Lucky13 (2013): TLS CBC — timing side-channel
  • ASP.NET Padding Oracle: .NET ViewState / encrypted forms auth
  • Java PKCS#1 oracle: Various JCE implementations

Defense

Use authenticated encryption. AES-GCM authenticates the ciphertext before decrypting:

# Python — AES-GCM (authenticated)
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
key = AESGCM.generate_key(bit_length=256)
aesgcm = AESGCM(key)
nonce = os.urandom(12)
ct = aesgcm.encrypt(nonce, plaintext, None)      # includes auth tag
pt = aesgcm.decrypt(nonce, ct, None)             # raises if tampered

With GCM, any modification to the ciphertext causes MAC verification to fail — the server never attempts decryption at all. There is no "valid vs invalid padding" signal to exploit.

Never use AES-CBC for cookie encryption or session tokens. Use AES-GCM, ChaCha20-Poly1305, or a high-level library like nacl/libsodium.
