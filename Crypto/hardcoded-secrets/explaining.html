Hardcoded Secrets in Source Code

What is it?

Hardcoded secrets are credentials, API keys, tokens, or cryptographic keys written directly into source code or configuration files. Once committed to version control or shipped in a build artifact, they are effectively public — anyone with repo access or who can view the built files can read them.

How secrets end up in code

  1. Developer convenience: "I'll replace this before merging" (they don't)
  2. Build pipeline injection: CI injects process.env.SECRET into the frontend bundle
  3. Copy-paste from docs: Working examples with real credentials pasted into code
  4. Config files committed: .env, config.json, secrets.yaml accidentally tracked by git

Common secret patterns

sk_live_*              Stripe secret key
AKIA[0-9A-Z]{16}       AWS Access Key ID
AIzaSy[A-Za-z0-9_-]+  Google API key
ghp_*                  GitHub personal access token
eyJ*                   JWT (base64-encoded)
-----BEGIN RSA-----    Private key

The git history problem

Removing a secret from the current codebase is not enough. Git permanently stores every commit. Anyone with repo access can find the secret in history:

git log --all --full-history -- .env
git show <commit>:.env

If a secret was ever committed, assume it is compromised. Rotate it immediately.

Cleanup

# Remove from history (rewrites all commits — coordinate with team)
git filter-repo --path .env --invert-paths

# Or use BFG Repo Cleaner
bfg --delete-files .env

Detection tools

  • gitleaks — Pre-commit hook + CI scanning
  • trufflehog — Deep git history scanning
  • semgrep — Static analysis with secret patterns
  • git-secrets — AWS-focused pre-commit hook
  • GitHub Secret Scanning — Automatic scanning on push (built-in)

Defense

  1. .gitignore: Always add .env, secrets.<em>, credentials.</em>
  2. Pre-commit hooks: Block commits containing secret patterns
  3. Environment variables: Inject secrets at runtime, never in the build artifact
  4. Secret management: Use Vault, AWS Secrets Manager, or GCP Secret Manager
  5. Frontend rule: No secret ever belongs in a JS bundle. Period.
