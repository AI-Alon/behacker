Weak Password Hashing

What is it?

Storing passwords hashed with fast algorithms (MD5, SHA-1, SHA-256) — with or without a salt — is insecure. Modern GPUs can compute billions of these hashes per second, making offline brute-force attacks feasible against entire databases in hours.

The problem with fast hashes

  • MD5: ~60 billion/sec GPU — Instant (rainbow table)
  • SHA-1: ~20 billion/sec GPU — < 1 second
  • SHA-256: ~10 billion/sec GPU — < 1 second
  • bcrypt (cost=12): ~20,000/sec GPU — Years
  • argon2id: ~1,000/sec GPU — Decades

Attack types

Rainbow table attack

Pre-computed tables map millions of common passwords to their hashes. Looking up 5f4dcc3b5aa765d61d8327deb882cf99 → password takes microseconds.

Wordlist / dictionary attack

Tools like hashcat and john the ripper hash every word in a wordlist (rockyou.txt has 14M entries) and compare against stolen hashes. Common passwords crack in seconds.

Rule-based mutation

password → P@ssw0rd, pa$$word, password1! — hashcat applies thousands of mutation rules automatically.

Why salting alone isn't enough

A unique per-password salt prevents rainbow tables but doesn't prevent wordlist attacks against MD5/SHA-1. With a GPU, an attacker still tries billions of candidates per second per hash.

Correct solution

Use a password hashing function — slow by design, with built-in salting:

// Node.js — bcrypt
const bcrypt = require('bcrypt');
const hash = await bcrypt.hash(password, 12);          // ~300ms
const valid = await bcrypt.compare(password, hash);    // constant-time

// Node.js — argon2id (recommended)
const argon2 = require('argon2');
const hash = await argon2.hash(password, { type: argon2.argon2id });
# Python — argon2
from argon2 import PasswordHasher
ph = PasswordHasher()
hash = ph.hash(password)
ph.verify(hash, password)

Migration strategy

If you have MD5/SHA-1 hashes in production:

  1. On next login, re-hash the plaintext password with bcrypt/argon2
  2. Replace the old hash in the database
  3. After sufficient time, force password reset for accounts that haven't logged in
