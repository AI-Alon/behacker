Default Credentials — How It Works

What is it?

Manufacturers ship devices and software with documented factory-default credentials.

If an administrator never changes them, the device is accessible to anyone who looks up the defaults.

How an attacker finds targets

1. Search Shodan for exposed devices:
   shodan search "Server: mini_httpd" country:US
2. Look up the device model in a default password database
3. Try admin/admin, root/root, etc.
4. Access granted — often with full administrative control

Common default credential databases

  • [CIRT.net](https://www.cirt.net/passwords)
  • [Default Password DB](https://defaultcredentials.com)
  • Device manuals (publicly available on manufacturer websites)

Commonly affected devices

  • Home routers: admin/admin, admin/password
  • IP cameras: admin/12345, root/(blank)
  • NAS devices: admin/(blank), admin/admin
  • Printers: admin/admin, admin/1234
  • Industrial PLCs: admin/admin, user/user
  • Web admin panels: admin/admin, root/root

The Fix

For manufacturers:
  - Ship each device with a unique random credential printed on the label
  - Force credential change on first login — block all access until done
  - Disable remote access by default

For administrators:
  - Change default credentials immediately after deployment
  - Audit devices on your network for factory defaults
  - Use a network scanner (Nmap, Nessus) to identify exposed admin interfaces

Leaving defaults in place is equivalent to leaving the front door unlocked with the key under the mat.
