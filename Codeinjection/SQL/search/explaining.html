SQL Injection — Search Box

What is it?

SQL injection in a search field lets an attacker manipulate the underlying query — returning all records, or using UNION injection to exfiltrate data from unrelated tables. The LIKE operator is just as vulnerable as a direct equality check.

How it works

Normal search query — the search term is wrapped in % wildcards and inserted into LIKE:

SELECT * FROM products WHERE name LIKE '%keyboard%';

If we input the following in the search box: ' OR '1'='1, the resulting query will be:

SELECT * FROM products WHERE name LIKE '%' OR '1'='1%';

'1'='1' is always true, so the WHERE clause is always true. Every row in the products table is returned. The query effectively becomes:

SELECT * FROM products;

Result: all products returned regardless of search term.

A more powerful attack — UNION injection to leak another table. Input: ' UNION SELECT id,username,email,password FROM users--

Resulting query:

SELECT * FROM products WHERE name LIKE '%' UNION SELECT id,username,email,password FROM users--;

The -- starts a comment, discarding the trailing %'. The UNION appends a second SELECT from the users table. The app receives user rows and displays them as if they were search results.

Result: full users table leaked — including passwords.

How the single quote breaks the string literal:

  • Intended: LIKE '% <term> %' — safe data inside the string
  • Injected: LIKE '% ' OR ... -- %' — quote closes the string, everything after is SQL

The fix: use a parameterized query.

SELECT * FROM products WHERE name LIKE ?
-- value passed: "%keyboard%"
-- The % wildcards are part of the bound value, never part of the SQL text.

Real-world impact

  • Data exfiltration: UNION injection leaks any table — users, payments, sessions, PII
  • Enumeration: Determine DB schema via information_schema.tables
  • Blind SQLi: Even without output, boolean/time-based techniques can extract data character by character
  • CVE-2008-4109: OpenSSH SQL injection in PAM module via search field

Escalation: dumping the schema
