Resource Exhaustion — How It Works

What is it?

Resource exhaustion DoS occurs when a server has no limits on input size or request rate, allowing a single attacker to consume all available CPU, memory, or disk — denying service to legitimate users.

No size limit

// Vulnerable: reads entire body with no limit
app.post("/api/process", express.text({ limit: "500mb" }), (req, res) => {
  const words = req.body.split(/\s+/);   // allocates O(n) memory
  processAllWords(words);                // O(n) CPU time
  res.json({ wordCount: words.length });
});

A single POST with 500 MB of data allocates 500 MB+ of memory and blocks the Node.js event loop for the entire processing time.

Common exhaustion targets

  • JSON parsing: Deeply nested JSON — parser stack overflows
  • XML parsing: Billion laughs attack — exponential entity expansion
  • Image processing: Decompression bomb — small file expands to gigabytes
  • Text search: Huge haystack — O(n) search per request
  • PDF/Office generation: Large input fills disk with generated files
  • Hash computation: bcrypt with very high work factor — forced by attacker

The Fix

// 1. Enforce request body size limit
app.use(express.json({ limit: "1mb" }));
app.use(express.text({ limit: "1mb" }));

// 2. Stream-process large inputs — don't buffer the whole thing
// 3. Move CPU-heavy work to worker threads / job queue
const { Worker } = require("worker_threads");

// 4. Rate limit by IP
const rateLimit = require("express-rate-limit");
app.use("/api/process", rateLimit({ windowMs: 60_000, max: 10 }));

// 5. Set request timeouts
server.setTimeout(5000);  // abort requests taking > 5s
