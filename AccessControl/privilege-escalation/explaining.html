Privilege Escalation — How It Works

What is it?

Privilege escalation occurs when a lower-privileged user can access functionality or data reserved for higher-privileged users — because the server only checks that a session exists, not what role the session has.

The vulnerability

// Vulnerable: only checks login state
app.get("/api/admin/users", (req, res) => {
  if (!req.session.user) return res.status(401).send("Unauthorised");
  // Missing: if (req.session.user.role !== "admin") return res.status(403).send("Forbidden");
  res.json(getAllUsers());
});

How attackers find hidden endpoints

1. Inspect JavaScript source code — API calls often reveal endpoint paths
2. Intercept traffic with Burp Suite — watch what the app calls in the background
3. Read API documentation leaked in /api/docs, /swagger, /openapi.json
4. Directory brute-force — gobuster / ffuf with a wordlist
5. Google dorking — site:target.com "api/admin"

Types of privilege escalation

  • Vertical: User → Admin (higher privilege level)
  • Horizontal: User A → User B (same level, different account)
  • Context-based: Exploit role in one context to gain access in another

The Fix

// Middleware that enforces admin role on every admin route
function requireAdmin(req, res, next) {
  if (!req.session.user)                       return res.status(401).json({ error: "Login required" });
  if (req.session.user.role !== "admin")       return res.status(403).json({ error: "Forbidden" });
  next();
}

// Apply to every admin route
app.use("/api/admin", requireAdmin);
app.get("/api/admin/users",   getUsers);
app.get("/api/admin/secrets", getSecrets);

Never rely on hiding URLs — always enforce authorisation server-side on every request.
